Vue封装组件时，嵌套组件的数据通信

本文主要介绍封装组件时嵌套组件的通信

场景

  在平时开发中，我们可能需要点击弹窗，点击弹出表单……

  如果我们将所有的弹窗、表单、图标……全部写在一个文件内，别说别人维护，就算是自己，过两个星期后回头一看也是懵逼的，虽然在阅读代码后始终可以着手维护，但是这种效率是很低的，并且会让维护者很抗拒看你的代码。

  所以我们可以从分离代码开始，把每个弹窗、每个表单都可以分为一个组件，这样不仅可以让代码看起来简洁更加易懂，无形中也对业务代码进行了解耦，更加方便以后的维护和需求的添加。

分离组件后的问题

  1. 子组件需要用父组件的数据
  2. 子组件需要改变父组件传递下来的数据
  3. 子组件内还嵌套着其它组件并且也需要顶层组件的数据
  4. 孙子组件需要父组件的数据，但子组件不需要
  5. 父组件内的各种组件需要调用父组件的方法

provide 和 inject 主要为高阶插件/组件库提供用例。并不推荐直接用于应用程序代码中。

这对选项需要一起使用，以允许一个祖先组件向其所有子孙后代注入一个依赖，不论组件层次有多深，并在起上下游关系成立的时间里始终生效。如果你熟悉 React，这与 React 的上下文特性很相似。

例如在3层组件中，组件1传递3个参数下组件2，但是组件2只需要其中的2个参数，还有一个参数是交给组件2内的组件3。

$attrs 官方文档

包含了父作用域中不作为 prop 被识别 (且获取) 的特性绑定 (class 和 style 除外)。当一个组件没有声明任何 prop 时，这里会包含所有父作用域的绑定 (class 和 style 除外)，并且可以通过 v-bind=”$attrs” 传入内部组件——在创建高级别的组件时非常有用。

解决问题5 (父组件内的各种组件需要调用父组件的方法)
$listeners 官方文档

包含了父作用域中的 (不含 .native 修饰器的) v-on 事件监听器。它可以通过 v-on=”$listeners” 传入内部组件——在创建更高层次的组件时非常有用。
