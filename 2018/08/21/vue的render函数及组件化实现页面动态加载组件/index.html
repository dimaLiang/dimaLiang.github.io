vue的render函数及组件化实现页面动态加载组件

渲染函数 & JSX

基础

Vue 推荐在绝大多数情况下使用 template 来创建你的 HTML。然而在一些场景中，你真的需要 JavaScript 的完全编程的能力，这就是 render 函数，它比 template 更接近编译器。

使用场景

当需要做一个统计页、首页时，我们可能需要根据列表菜单或权限来渲染不同的组件，这时我们可以通过将一个个模块封装成一个组件，通过v-if,v-else进行渲染出匹配的组件。

当然以上的方法也可以实现，但是更加蛋疼的是我们可能需要根据后台返回的模块顺序来渲染组件的排序，允许用户更改模块位置，自定义列表样式……这时v-if,v-else就不能够满足我们的需求了。

本次分享示例主要利用后端返回的权限码进行匹配组件，并根据后台返回的顺序渲染页面。

以上的方法渲染顺序应该为先渲染 tem3 再渲染 tem1 组件。

但是问题来了，如果由组件内的mounted周期请求数据，那么一下子会有很多请求并发，所以我们在请求权限时要求后端把各组件的数组同时返回。这样我们就可以在父组件为不同的子组件分配不同的数据，通过props传递子组件渲染数据。

这样就大概完成了通过后台返回的权限码及数据匹配相应的组件后动态的渲染组件。

但是人是不会满足的，对于非常潮的前端工程师，我必须跟上潮流，发型要37分，页面也要37分，所以我们可以在定义每个模块对应的权限时候给他一个37分的标签。

最后一点实现动态就是使用watch监控菜单选项，菜单每次改变则根据菜单值来请求后端得到相应的authCode及数据。
